История релизов бота @tychyna_images_bot.

Версионирование: простой инкремент 0.X (без semver — внутренний бот).
Перед каждым деплоем-с-уведомлением: bump в `VERSION`, новый файл

`HF_CONCURRENCY=2 → 4` в .env на проде. Генерация в 2 раза быстрее.

Один юзер (друг) — лимит OpenAI rate-limit держит. Если потом

## 0.4 — 2026-05-26

**Разделение ролей рефов: inline = композиция, folder = стиль.**

Друг: «у меня есть точная по композиции картинка, которую нужно
завернуть в мой стиль. Если есть картинка-пример — прям повторяй».

До 0.4 промпт говорил «не копируй композицию из рефов» — это
блокировало точное повторение. Теперь:
- Если есть inline-реф из .docx → `COMPOSITION_HINT_FOR_INLINE`:
  «первая картинка = blueprint, copy composition/layout/perspective
  precisely»
- Если есть И folder-рефы → дополнительно `STYLE_HINT_WHEN_INLINE_AND_FOLDER`:
  «style components from OTHER refs, composition only from first»
- Если только folder → старый `STYLE_HINT_FOR_REFS` (не копировать subject)

Также: в режиме «✏️ Внести правки» теперь можно прислать фото с
подписью — `on_ref_photo` детектит `awaiting_edit`, скачивает фото
в `/tmp/edit-refs/`, передаёт как `extra_ref_path` в `_do_edit`.
Caption становится инструкцией (или дефолт).

## 0.3 — 2026-05-26

Inline-рефы из .docx переключены с opt-OUT на **opt-IN**.

Раньше каждая встроенная картинка автоматически шла как референс в
следующий `Рис.` Это давало false-positives: декоративные картинки
становились рефами и портили генерацию.

Теперь:
- По умолчанию **не привязываем**
- Триггер по ключевой фразе в title слота: «выше», «из картинки»,
  «с картинки», «за референс», «по референсу», «использовать картинку»
- `Рис.[ref]` форсит привязку даже без триггера
- `Рис.[noref]` явно отказывается даже при триггере (+ не сохраняем
  как папку-категорию)

Также: формат broadcast-сообщений стал короче.

## 0.2 — 2026-05-26

**Большая фича — бот видит картинки в .docx:**
- При парсинге docx все встроенные base64 картинки извлекаются в `/tmp/article-img-bot-inline-refs/`, в текст ставится `[INLINE_IMAGE:path]`
- Парсер ассоциирует inline-картинку с ближайшим следующим маркером `Рис.`
- `ImageSlot.inline_refs` хранит пути
- Worker (OpenAI `images.edit`) использует inline-рефы с приоритетом над `refs/<категория>/`
- Krea пока без inline-рефов (нужен upload-хост)

**Парсинг таблиц:**
- Loader перешёл с `mammoth.convert_to_markdown` на `convert_to_html` — markdown терял структуру таблиц
- HTML-таблицы конвертируются в markdown-syntax (`| col | col |`) с разделителем
- Парсер видит таблицу под `Рис.` → каждая строка становится буллетом

**UX:**
- Caption файла: `📄 имя · X из N` с подсказкой на первой картинке
- В смете: `📎 #N — есть inline-рефы` или `⚠️ #N — ссылка на картинку, но не нашёл`

## 0.1 — 2026-05-26

Первый релиз с уведомлением. До этого было ~2 недели разработки
(см. git log от 2026-05-19 до 2026-05-26).

**Новое:**
- 4 модели на выбор в смете: GPT-Image (OpenAI), Nano Banana Pro, Flux Pro, Ideogram 3.0 (всё через Krea-аггрегатор)
- Стрим картинок по одной с кнопками ✅ ОК / ✏️ Внести правки
- `/refs → ✏️ Настроить стиль` — текстовое описание стиля влияет на промпт
- Маркеры `Рис.[grid]` / `Рис.[scene]` — явный override типа лейаута

**Починили:**
- Стиль рефов теперь копируется как эстетика, не буквально (промпт декомпозирует на палитру/материалы/фон, без копии сюжета)
- Порог инфографики поднят с 2 до 4 буллетов
- Логи в systemd journal через `logging.basicConfig`
- `running=True` сбрасывается при старте бота (был залип после OOM)
- Кэш убран — каждый прогон свежий
- `images.edit` для OpenAI вместо `images.generate` когда есть рефы
- `_clear_awaiting` чинит конфликты `awaiting_*` флагов между обработчиками
- `concurrent_updates=True` — тапы не блокируются длинной генерацией
- Параллельная работа нескольких юзеров одновременно
