# Статья → картинки (Telegram-бот)

Бот принимает статью (файл `.docx`/`.md`/`.txt` или ссылку), находит
в ней задания на картинки под маркером **«Рис.»**, генерит их по
референсному стилю через Higgsfield и отдаёт **ZIP-архивом**.

Полное ТЗ — [`TZ.md`](./TZ.md). Это якорь, любые изменения сверять с ним.

## Как работает

```
статья → парсер «Рис.» → классификатор (инфографика/сюжетная)
       → промпт-билдер (рефы по типу) → Higgsfield-воркер (кэш+retry)
       → постобработка размера → ZIP → Telegram
```

Ядро не зависит от Telegram (тонкая обёртка) и от наличия оплаты:
режим **MOCK** гоняет весь конвейер без вызовов API и без затрат.

## Быстрый старт (локально, MOCK, бесплатно)

```bash
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
cp .env.example .env            # HF_MODE=MOCK по умолчанию

# демо-прогон на примере статьи 8Blocks:
.venv/bin/python run_local.py tests/fixtures/8blocks_excerpt.txt

# тесты:
.venv/bin/python -m pytest -q
```

В MOCK картинки — заглушки (видно весь путь и кэш), доллары не тратятся.

## Прод (REAL, платно)

Нужно от заказчика (вилка «по-дружески vs платно», TZ 8.5):

1. **Higgsfield Creator** — платный тариф, ключ `KEY_ID:KEY_SECRET`.
2. **Токен бота** от @BotFather.
3. **Список user_id** тех, кому можно (иначе чужие жгут бюджет).
4. **Рефы** в `refs/infographic/` и `refs/story/` (заказчик заливает).

`.env`:

```
HF_MODE=REAL
HF_CREDENTIALS=KEY_ID:KEY_SECRET
TELEGRAM_TOKEN=...
HF_ALLOWED_USER_IDS=12345678,987654   # пусто = бот не пустит НИКОГО
```

Запуск: `python -m src.telegram_bot`

## Хостинг (минимум денег — TZ 7.7г)

Платный API неизбежен, но **хостинг должен быть бесплатным/дешёвым**:

- бесплатные tier: Oracle Cloud Always Free, fly.io, Railway;
- либо уже имеющийся VPS — `Dockerfile` делает деплой одинаковым везде:
  `docker build -t artbot . && docker run --env-file .env artbot`.

Не поднимать ради этого новый платный VPS.

## Важные ограничения (честно)

- **`.docx` конвертится через `textutil` — только macOS.** На Linux/в
  контейнере его нет: в проде вход — **Google Doc-ссылка** (чистый
  txt-экспорт) или `.md`/`.txt`. Бот про это скажет внятной ошибкой.
- **Приватный Notion** вне scope (нужен интеграция-токен заказчика).
  Публичные `notion.site` работают как обычная веб-страница.
- **REAL-ветка Higgsfield SDK** написана по официальному SDK, но
  документация скудная (TZ 6) — точную форму вызова надо **сверить на
  живом оплаченном аккаунте**. До этого REAL не проверялся.

## Экономия (встроено)

- кэш: одинаковый блок при тех же рефах не генерится повторно;
- смета **до** запуска — бот пишет «N генераций ≈ $X» и ждёт `/go`;
- 1 вариант на слот; сменили рефы → перегенерация (старое из кэша не отдаётся).

## Статус

Готово и протестировано (MOCK, без затрат): парсер, классификатор,
промпт-билдер, воркер, постобработка, pipeline, whitelist, бот,
загрузчик файлов и ссылок.
